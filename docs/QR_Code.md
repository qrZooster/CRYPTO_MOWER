# QR_CODE: ПРАВИЛА НАПИСАНИЯ КОДА PYTHON VERSION


## Правило #1 **Правило минимального конструктора (Tradition Core QR_Code):**

1. В `__init__()` класса мы принимаем только те аргументы, без которых объект в принципе не может существовать корректно (то есть он не может быть создан в рабочем состоянии вообще).

   * Примеры такого уровня: ссылка на обязательного владельца/контекст, идентификатор без которого его нельзя зарегистрировать в системе, путь к ресурсу без которого он упадёт и т.д.
   * Всё остальное — НЕ в параметры конструктора.

2. Все настраиваемые свойства (заголовки, тексты, цвета, режимы отображения панелей, любые визуальные и поведенческие флаги и т.п.) должны устанавливаться уже после создания объекта как поля инстанса.

   * Объект всегда создаётся в валидном дефолтном состоянии сам.
   * Пользователь потом просто правит поля: `card.title = "..."`, `card.panel_footer = False`, и т.д.
   * Мы НЕ заставляем пользователя прокидывать в `__init__` 10 аргументов, чтобы просто «получить нормальную карту».

3. Конструктор — это не форма на 20 полей.
   Это точка, которая гарантирует "я живу в дереве и не ломаю приложение".
   Всё, что касается внешнего вида, поведения, цветов, текстов, стилей, загрузок данных — это уже конфигурация после `super().__init__()`.

Короче:

> **В параметры конструктора попадает только то, без чего объект невозможно корректно создать. Всё остальное — свойства после создания.**
