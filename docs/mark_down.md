## Can I add my **own** icons?
Short answer: **pure Markdown can’t define new emoji**, but you have great options:

### 1) Use images (PNG/SVG)
```markdown
![Trash](icons/trash.svg)
![Logo](https://example.com/logo.png)
```
Tip: for sizing, use HTML (works in GitHub READMEs too):
```html
<img src="icons/trash.svg" alt="Trash" width="18" height="18">
```

### 2) Inline SVG (portable, crisp)
```html
<svg width="18" height="18" viewBox="0 0 24 24" aria-hidden="true">
  <path d="M3 6h18M8 6V4h8v2m-1 0v14a2 2 0 0 1-2 2H9a2 2 0 0 1-2-2V6h10z"/>
</svg>
```

### 3) Badges (shields.io)
```markdown
![build](https://img.shields.io/badge/build-passing-brightgreen)
![version](https://img.shields.io/badge/version-1.2.3-informational)
```

### 4) Icon packs (web pages)
In Markdown that’s rendered as **HTML pages**, you can include icon fonts or SVG sets (e.g., Font Awesome, Tabler) and then use `<i>` or `<svg>` tags. Not available in plain GitHub README without extra HTML/CSS.

> Platforms like **Slack/Discord** allow custom emoji uploads; **GitHub** does **not** (outside of using images/SVG in your repo).
